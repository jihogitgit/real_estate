## 알려진 이슈 및 주의사항

### 버튼 컴포넌트 asChild 미지원
`src/components/ui/button.tsx`의 Button은 `asChild` prop을 지원하지 않음 (shadcn/ui가 Radix 아닌 @base-ui/react 기반).
- **올바른 패턴:** `<a className={cn(buttonVariants({ variant: 'default' }))} href="...">` 또는 `<Link className={cn(buttonVariants(...))} href="...">`
- **잘못된 패턴:** `<Button asChild><a>...</a></Button>` — plan 문서 일부 단계에 이 패턴이 있으나 **무시할 것**

### zsh 대괄호 경로 git add 주의
zsh에서 `[slug]`, `[id]`, `[name]` 포함 경로는 반드시 따옴표로 감쌀 것:
```bash
git add "src/app/apply/[id]/page.tsx"
git add "src/app/news/[slug]/page.tsx"
```

### Node.js 버전
항상 v22 사용:

source ~/.nvm/nvm.sh && nvm use 22

### git push 금지

### Sentry 위저드 interactive CLI
Task 27에서 Sentry 위저드는 interactive하게 DSN을 물어봄. Claude가 직접 실행 불가.
사용자가 직접 실행 후 생성된 config 파일 확인 및 수정 필요.

### Supabase 테이블 필요 (런칭 전)
다음 테이블이 Supabase DB에 없으면 Task 25 기능이 런타임 에러:
```sql
-- saved_apartments 테이블
CREATE TABLE saved_apartments (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  apartment_id text NOT NULL,
  created_at timestamptz DEFAULT now(),
  UNIQUE(user_id, apartment_id)
);

-- alerts 테이블
CREATE TABLE alerts (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  apartment_id text NOT NULL,
  alert_days_before int NOT NULL DEFAULT 1,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now(),
  UNIQUE(user_id, apartment_id, alert_days_before)
);

-- RLS 활성화
ALTER TABLE saved_apartments ENABLE ROW LEVEL SECURITY;
ALTER TABLE alerts ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users own their saves" ON saved_apartments FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users own their alerts" ON alerts FOR ALL USING (auth.uid() = user_id);
```
